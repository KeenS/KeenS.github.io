  • プログラミング言語の理論 ≒ λ計算
  • そもそもλ計算はメモリのことを考慮してない
    • GCがあるかのように記述される

===

可能性の議論

  • λ計算にメモリ管理まで含めて理論を立てたものは存在する（静的なメモリ管理）
    • リージョンというものをベースにしている 参考
  • さらにリージョンを自動で推論する理論もある（強いメモリ管理）
    • リージョン推論という

===

軽くリージョンの話

  • 型と同じようにプログラム全体を伝わるメタデータ。
  • データが保存される場所を指す。リージョンはいくつもある。
  • リージョン推論でデータがどのリージョンに入るかが分かる
  • さらにリージョンのサイズもある程度予想がつくので静的に管理出来る
  • 関数などは引数のリージョンに対して多相になる「リージョン多相」などもある
  • この辺はMartin Elsmanの論文たちを参考にして下さい
    • A Brief Introduction to Regionsとか。

Rustのライフタイムと所有権

===

Cの例

{
    int *x = malloc(sizeof(int));

    // we can now do stuff with our handle x
    *x = 5;

    free(x);
}

===

Rustに翻訳

{
    let x = Box::new(5);
}

===

少しいじってみる

trivialに見える

fn main() {
    let x = Box::new(5);

    add_one(x);
}

fn add_one(mut num: Box<i32>) {
    *num += 1;
}

===

少しいじってみる

printlnを追加してみる

fn main() {
    let x = Box::new(5);

    add_one(x);

    println!("{}", x);
}

fn add_one(mut num: Box<i32>) {
    *num += 1;
}

===

少しいじってみる

エラーになる。

error: use of moved value: `x`
   println!("{}", x);
                  ^

===

所有権

add_one を呼んだ時点で所有権が add_one に移るので println! では使えない。

fn main() {
  let x = Box::new(5);

  add_one(x);

  println!("{}", x);
}

fn add_one(mut num: Box<i32>) {
    *num += 1;
}

===

所有権

新たに値を返してもらえば使える。

fn main() {
  let x = Box::new(5);

  let y = add_one(x);

  println!("{}", y);
}

fn add_one(mut num: Box<i32>) -> Box<i32> {
  *num += 1;

  num
}

===

所有権の貸し借り

  • さっきの例は面倒。
  • add_one が x を奪ったのが問題。
  • x を「借り」ることが出来る。

===

所有権の貸し借り

fn main() {
  let mut x = 5;

  add_one(&mut x);

  println!("{}", x);
}

fn add_one(num: &mut i32) {
  *num += 1;
}

===

ライフタイム

先の add_one はライフタイム(リージョン)アノテーションを省略していた。
省略せずに書くとこうなる。(リージョン多相)

fn add_one<'a>(num: &'a mut i32) {
  *num += 1;
}

===

ライフタイム

スコープの終わりでライフタイムが終わる。

fn main() {
  let y = &5;     // -+ y goes into scope
                  //  |
  // stuff        //  |
                  //  |
}                 // -+ y goes out of scope

===

LTの明示的宣言

こんな構造体を宣言したとする。 x はコンストラクタに渡された値のライフタイムを引き継ぐ。

struct Foo<'a> {
    x: &'a i32,
}

===

LTの明示的宣言

f.xのライフタイムが y のライフタイムに制限されるので y より広いスコープにある x には代入出来ない。

fn main() {
  let x;                  // -+ x goes into scope
                          //  |
  {                       //  |
    let y = &5;           // ---+ y goes into scope
    let f = Foo { x: y }; // ---+ f goes into scope
    x = &f.x;             //  | | error here
  }                       // ---+ f and y go out of scope
                          //  |
  println!("{}", x);      //  |
}                         // -+ x goes out of scope
