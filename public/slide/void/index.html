発散型

  • !
  • ドキュメント
    • 標準ライブラリだと exit とか
    • 他には無限ループ、パニック、 return`/continue` など
  • 呼び出し元に制御が返らない
  • どんな型にもなれる

===

発散型

use std::process::exit;
// pub fn exit(code: i32) -> !

let hoge: i32 = exit(0);
let fuga: () = exit(0);

===

match と型

  • match の返り値は腕の型で決まる
  • 発散型と他の型の組み合わせだと他の型になる
let piyo: i32 = match 0 {
    // !
    0 => exit(0),
    // i32
    n => 1i32,
};

===

Void

enum Void {}

===

Void

  • バリアント(=コンストラクタ)がない
    • 値を作れない
  • もしその値を受け取れるとしたらどうなる？

===

Void の作り方

  • 発散型を使うと(コード上は)作れる
  • 発散型なので実行時にはこのコードに到達しない
let void: Void = unimplemented!();
//...

===

Void の作り方2

  • 関数を使うと(コード上は)受け取れる
  • 実際にはこの関数は呼べない
fn take_void(void: Void) {
    // ...
}

===

Void にマッチ

  • Q: これはコンパイルが通る？
  • A: 通る
fn absurd(void: Void) {
    match void {};
}

===

Void にマッチ

  • 返り値型はなんでもいい
fn absurd(void: Void) -> () {
    match void {}
}

===

Void にマッチ

  • 返り値を発散型にもできる!
fn absurd(void: Void) -> ! {
    match void {}
}

===

Void と列挙型

  • Result<T, Void> という型を考える
  • Void は作れないので Err はありえない
  • Ok しか存在しない Result 型に

===

Void と列挙型

  • Q: これはコンパイルが通る？
  • A: 通らない
let result: Result<(), Void> = Ok(());
match result {
  Ok(()) => //
}

===

absurd の使いどころ

  • これはコンパイルが通る
let result: Result<(), Void> = Ok(());
match result {
  Ok(()) => //
  Err(void) => absurd(void),
}

===

absurd の使いどころ

  • こういうのも作れる
fn safe_unwrap<T>(result: Result<T, Void>) -> T {
    match result {
        Ok(t) => t,
        Err(void) => absurd(void),
    }
}

===

まとめ

  • ! は特殊な型だよ
  • 空の列挙型 (Void) から ! が作れるよ
